# Гайд для менеджера: тексты, PDF, голосовое и картинки

Этот бот сделан так, чтобы **менять контент без программиста**. Вы трогаете
только два места:

1. **Тексты, ссылки, дата вебинара** → файл `src/content/ru.json`
2. **Картинки, PDF, голосовое** → папка `src/assets/`

Никакие `.js`-файлы и `.env` для смены контента открывать не нужно.

> **Где работает бот:** на сервере Render — круглосуточно, без вашего ноутбука и
> без ngrok. Адрес: `https://simple-webinar-bot.onrender.com`
> (проверка работоспособности: `https://simple-webinar-bot.onrender.com/health`).
> После правок в `ru.json`/картинках изменения подхватываются при следующем
> перезапуске сервиса на Render. Технические детали деплоя — в `README.md`.

## 1. Куда добавить PDF (подарок за регистрацию)

Есть **два способа**. Можно использовать любой — выберите, что удобнее.

### Способ А — прислать сам файл PDF (рекомендуется)

1. Назовите файл **точно** `gift.pdf`.
2. Положите его в папку `src/assets/`.
3. Готово. Теперь подарок приходит человеку **файлом PDF**:
   - автоматически в воронке (через ~20 секунд после регистрации);
   - и по команде `/подарок`.

Чтобы поменять PDF позже — просто замените файл `src/assets/gift.pdf` на новый
с тем же именем. Перезапуск бота не нужен (файл читается в момент отправки).

> Хотите другое имя файла? Поменяйте его в `src/content/ru.json`:
> секция `"assets"` → `"giftPdf": "gift.pdf"` — впишите своё имя и положите файл
> с этим именем в `src/assets/`.

### Способ Б — дать ссылку на PDF (если файл лежит на сайте/в облаке)

1. Откройте `src/content/ru.json`.
2. Найдите секцию `"leadMagnet"` → поле `"pdfUrl"`.
3. Впишите ссылку:
   ```json
   "leadMagnet": {
     "pdfUrl": "https://ваш-сайт.ru/podarok.pdf"
   }
   ```

**Что важно:** если файла `src/assets/gift.pdf` нет, бот автоматически пришлёт
текст со **ссылкой из `pdfUrl`**. Так что бот работает в любом случае — с файлом
или со ссылкой.

## 2. Куда добавить голосовое от лектора (День 4 прогрева)

По ТЗ на **4-й день прогрева** приходит **голосовое сообщение от лектора**.
Оно хранится там же, где картинки — в `src/assets/` — и легко меняется.

1. Запишите голосовое (например, в Telegram «себе в Избранное», затем сохраните
   файл; или запишите на телефон).
2. Назовите файл **точно** `lecturer-voice.ogg`.
3. Положите его в папку `src/assets/`.
4. Готово. На 4-й день прогрева бот отправит это голосовое каждому
   зарегистрированному пользователю.

Чтобы поменять голосовое позже — просто замените файл
`src/assets/lecturer-voice.ogg` на новый с тем же именем.

### Про формат файла

- **`.ogg`** (формат OPUS) — приходит как настоящее **голосовое сообщение**
  (кружок-голосовое в Telegram). Это рекомендуемый вариант.
- **`.mp3`** тоже поддерживается, но приходит как **аудио-трек** (плеер), а не
  голосовой кружок. Если используете mp3 — поменяйте имя в `ru.json`
  (`"assets"` → `"lecturerVoice"`) на `lecturer-voice.mp3`.

### Если голосового пока нет

Если файла нет, бот **не сломается** — на 4-й день он пришлёт обычный текст со
ссылкой из `src/content/ru.json` (`"links"` → `"audio"`). Когда добавите файл,
бот начнёт присылать именно голосовое.

> Как это устроено внутри: текст 4-го дня прогрева задан в `ru.json`
> (`messages.warmup`, четвёртое сообщение в списке). К нему привязано голосовое
> `lecturer-voice.ogg`. Менять порядок сообщений в списке не нужно.

## 2.5. Тайминги: дата вебинара и расписание прогрева

Где что меняется:

| Что поменять                                  | Где                                                       | Кто      |
| --------------------------------------------- | --------------------------------------------------------- | -------- |
| **Дата и время вебинара**                     | `src/content/ru.json` → `webinar.date`, `webinar.time`    | менеджер |
| Часовой пояс                                  | `src/content/ru.json` → `webinar.timezone`                | менеджер |
| **Тест-режим** (всё приходит за секунды)       | `.env` → `SCHEDULE_TEST_MODE`                              | менеджер |
| Тайминги прогрева/welcome/follow-up (в днях)   | `src/flows/scheduler.js` → блок `realSchedule`            | разработчик |
| Напоминания в день эфира (за 3ч, 15м, 2м…)     | `src/flows/scheduler.js` → `scheduleLiveDay`             | разработчик |

### Тест-режим — почему прогрев «срабатывает очень быстро»

В `.env` есть переключатель:

```env
SCHEDULE_TEST_MODE=false   # боевой режим: прогрев по дням (День 1–5), напоминания от даты вебинара
SCHEDULE_TEST_MODE=true    # тест: ВСЕ сообщения приходят за секунды (для проверки воронки)
```

Если прогрев приходит через секунды, а не по дням — значит включён `true`.
Для реального запуска должно быть **`false`** (уже выставлено).

### Как работает боевое расписание (`SCHEDULE_TEST_MODE=false`)

- **После регистрации (Блок 0):** 0.1 через 10 сек, 0.2 (подарок) через 3 мин, 0.3 (вопрос) через 10 мин.
- **Прогрев (Блок 1):** одно сообщение в день — День 1, 2, 3, 4, 5 после регистрации.
  Если какой-то день уже позже даты вебинара — это сообщение не отправляется.
- **День эфира:** напоминания считаются от `webinar.date` + `webinar.time`
  (за 10 ч, за 3 ч, за 15 мин, за 2 мин, +10 мин, +30 мин).
- **После вебинара (Блок 3):** серия дожима на группу/диагностику.
  - Кто **был** на вебинаре: День 0 (через 15 мин) → Дни 1, 2, 3, 4, 5, 7.
  - Кто **не пришёл**: День 0 — запись вебинара, дальше та же серия (Дни 1–7).
  - Тексты — в `src/content/ru.json` → `messages.followUp` (для пришедших) и
    `messages.noShowFollowUp` (для не пришедших).
  - **Важно:** эта серия запускается, когда админ после эфира выполняет
    синхронизацию посещаемости Zoom (`/sync_zoom_attendance`) — именно она
    определяет, кто был, а кто нет, и ставит нужную последовательность.

> Дни прогрева и сдвиги напоминаний заданы в `src/flows/scheduler.js` в блоке
> `realSchedule` (с понятными комментариями). Их меняют редко и обычно силами
> разработчика — а вот **дату/время вебинара** и **тест-режим** менеджер меняет
> сам в `ru.json` и `.env`.

После смены `SCHEDULE_TEST_MODE` или даты вебинара **перезапустите бота**.

## 2.6. Оффер в эфире — команда `/send_offer`

В день вебинара напоминания (утро, −3 ч, −15 мин, −2 мин, +10 мин, +30 мин)
уходят **автоматически** — они привязаны к дате/времени вебинара из `ru.json`.

А вот **синхронный оффер** («Лектор только что рассказал о нашем оффере…»)
отправляется **вручную в один клик — ровно в тот момент**, когда лектор
объявляет оффер на вебинаре.

**Как отправить (админ):**
1. Открой панель оффера — двумя способами:
   - в меню бота нажми кнопку **«📢 Отправить оффер всем»** (она видна только админам), либо
   - напиши команду `/send_offer`.
2. Бот покажет подтверждение и сколько человек получат рассылку.
3. Когда лектор объявил оффер — нажми эту кнопку **один раз**. Бот разошлёт оффер
   **всем зарегистрированным** и покажет «✅ Оффер разослан: N получателям».
   Кнопка после нажатия пропадает — случайно отправить дважды нельзя.
   Если передумал — нажми «Отмена».

Текст оффера и ссылки меняются в `src/content/ru.json` → `messages.finalOffer`.

**Кто может это сделать:** только Telegram-аккаунты из `.env` → `ADMIN_TELEGRAM_IDS`
(ID через запятую). Обычный участник команду запустить не сможет. Чтобы узнать
свой Telegram ID, напишите боту `@userinfobot`. После смены `ADMIN_TELEGRAM_IDS`
перезапустите бота.

## 3. Картинки

5 картинок (обложки и баннеры) — в папке `src/assets/`. Полный список имён и где
что показывается смотрите в [`src/assets/README.md`](src/assets/README.md).

Принцип тот же: положите PNG с нужным именем — бот пришлёт картинку; нет файла —

## 3.5. Истории учеников (кейсы / отзывы)

Кейсы учеников вынесены в переиспользуемую структуру — **без имён в коде**.
Всё меняется только в `src/content/ru.json` и `src/assets/cases/`.

### 1. Заменить фото ученика
Положите PNG в `src/assets/cases/` с именем по номеру кейса: `1.png`, `2.png`, `3.png`.
Нет файла — кейс придёт текстом (бот не падает). См.
[`src/assets/cases/README.md`](src/assets/cases/README.md).

### 2. Заменить текст кейса
`src/content/ru.json` → секция `cases`. У каждого кейса поля:
`title` (заголовок), `name` (имя), `problem` (проблема), `solution` (что помогло),
`result` (результат), `quote` (отзыв). Меняйте текст после двоеточия.

Бот собирает из них сообщение в таком виде:

```
📌 История ученицы

👤 Елена

Проблема:
…

Что помогло:
…

Результат:
…

💬 "…"
```

### 3. Какой кейс где показывать
`src/content/ru.json` → секция `caseUsage`. Это просто «слот → номер кейса»:

```json
"caseUsage": {
  "warmupDay2": 1,
  "warmupDay5": 2,
  "followUpDay1": 1,
  "followUpDay4": 3,
  "followUpDay5": 2
}
```

- Слева — место в воронке (День прогрева / День follow-up).
- Справа — `id` кейса из `cases`.
- Чтобы показать другой кейс — поменяйте **только число**. Чтобы убрать кейс из
  слота — удалите строку (тогда там покажется обычное сообщение этого дня).
- Один и тот же кейс можно ставить в несколько слотов (как кейс `2` выше).

### 4. Никакого кода
Менять кейсы, фото и их расстановку можно полностью в `ru.json` и
`src/assets/cases/`. Код трогать не нужно.

## 4. Памятка: где что лежит

| Что меняем                              | Где                                                  |
| --------------------------------------- | ---------------------------------------------------- |
| PDF-подарок (файл)                      | `src/assets/gift.pdf`                                 |
| PDF-подарок (ссылка)                    | `src/content/ru.json` → `leadMagnet.pdfUrl`          |
| Голосовое от лектора (День 4)           | `src/assets/lecturer-voice.ogg`                       |
| Голосовое (запасной вариант — ссылка)   | `src/content/ru.json` → `links.audio`               |

| Ссылка на Zoom                          | `src/content/ru.json` → `webinar.zoomUrl`           |
| Юзернейм менеджера                       | `src/content/ru.json` → `manager.username`          |
| Все тексты сообщений                    | `src/content/ru.json` → `messages`                  |

Имена файлов в `src/assets/` заданы в `src/content/ru.json` → секция `"assets"`.

## 5. Главное правило

- Файла **нет** → бот присылает **текст** (со ссылкой). Бот не падает никогда.
- Файл **есть** → бот присылает **файл** (PDF / голосовое / картинку).
- Заменить контент = положить файл с тем же именем. Перезапуск не нужен.

После правок в `ru.json` проверьте, что файл не сломан (бот тоже проверяет это
при запуске и не стартует с понятной ошибкой, если чего-то не хватает):

```bash
node -e "import('./src/content/loadContent.js').then(()=>console.log('content OK')).catch(e=>{console.error(e.message);process.exit(1)})"
```
