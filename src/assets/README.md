# Картинки бота (src/assets)

Сюда кладутся изображения, которые бот отправляет вместе с сообщениями.
Имена файлов заданы в `src/content/ru.json` → секция `"assets"`.

Картинки (PNG):

| Файл                        | Где показывается                                   |
| --------------------------- | -------------------------------------------------- |
| `webinar-cover.png`         | команда `/вебинар` — обложка вебинара              |
| `pdf-cover.png`             | команда `/подарок` — обложка PDF-лидмагнита        |
| `diagnostics-banner.png`    | команда `/диагностика` — баннер диагностики        |
| `group-course-banner.png`   | команда `/группа` — баннер групповых занятий       |
| `registration-thank-you.png`| после регистрации в Zoom — «спасибо за регистрацию»|

Файл и голосовое:

| Файл                  | Где отправляется                                              |
| --------------------- | ------------------------------------------------------------ |
| `gift.pdf`            | подарок за регистрацию — приходит файлом (и по `/подарок`)    |
| `lecturer-voice.ogg`  | День 4 прогрева — голосовое от лектора                        |

Голосовое: формат `.ogg` (OPUS) приходит «голосовым кружком». `.mp3` тоже
работает, но показывается как аудио-трек. Имена файлов берутся из
`src/content/ru.json` → секция `"assets"`. Если файла нет — бот отправит текст
со ссылкой (см. `MANAGER_GUIDE.md`).

## Как заменить картинку

1. Подготовьте новый PNG (рекомендуется ширина 1080–1280 px).
2. Назовите файл **точно так же**, как в таблице выше.
3. Положите его в эту папку (`src/assets/`), заменив старый.
4. Перезапуск бота не обязателен — картинки читаются с диска при каждой отправке.

## Если картинки нет

Файл можно не класть вовсе. Если изображение отсутствует, бот **не падает** —
он отправит то же сообщение текстом и запишет предупреждение в лог:
`[assets] image not found: <файл>; sending text only`.

Так что бот работает и с картинками, и без них.

## Сменить имя файла

Если хотите другое имя файла — поменяйте его в `src/content/ru.json` в секции
`"assets"`, и положите файл с этим именем сюда.
